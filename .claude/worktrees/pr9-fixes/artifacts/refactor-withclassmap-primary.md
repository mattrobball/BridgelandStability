# Refactor: Make `WithClassMap C v` the Primary Type

## Ethos

This refactor is about getting the abstraction right, not about making
code compile. Every change must be motivated by mathematical clarity.
The question at each step is: "does this match how the mathematics
works?" — not "does this make Lean happy?"

**Rules:**
- Understand the mathematics first. Read the paper. Then write the code.
- Build at the natural level of generality — the level where the math
  lives. If a construction works for general `v`, define it for `v`.
- No local optimizations. No "patch it to compile" fixes. If a proof
  breaks, understand WHY before touching it.
- No bridge code. If two theorems don't compose, the types are wrong.
  Fix the types, don't add a bridge.
- Simplification is the metric. Fewer definitions, fewer lines, more
  mechanical proofs. If the codebase isn't simpler after a change, the
  change is wrong.

## Mathematical Context

### What Bridgeland actually defines

A **stability condition on `D` with respect to a lattice `Λ`** is a pair
`(Z, P)` where `P` is a locally finite slicing and `Z : Λ → ℂ` is the
central charge, satisfying compatibility: for nonzero `P(φ)`-semistable `E`,
`Z(v[E]) = m · exp(iπφ)` with `m > 0`, where `v : K₀(D) → Λ` is the
class map.

The **Bridgeland seminorm** on `Hom(Λ, ℂ)` is:

    ‖U‖_σ = sup { |U(v[E])| / |Z(v[E])| : E nonzero σ-semistable }

The **Bridgeland topology** on `Stab_Λ(D)` is generated by basis
neighborhoods:

    B_ε(σ) = { τ : ‖τ.Z - σ.Z‖_σ < sin(πε) and d(P_σ, P_τ) < ε }

This is the BLMNPS topology (Bayer–Lahoz–Macrì–Nuer–Perry–Stellari,
arXiv:1902.08184, §12.2): the coarsest making both the charge map
`σ ↦ σ.Z` and the slicing map continuous.

### The key theorems

**Theorem 1.2** (Bridgeland, refined by BMS16 Appendix A, Bayer 1606.02169):
For each connected component `Σ` of `Stab_Λ(D)`, define

    V(Σ) = { U : Λ → ℂ : ‖U‖_σ < ∞ }  (independent of σ ∈ Σ by Lemma 6.2)

The central charge map `Z : Σ → V(Σ)`, `σ ↦ σ.Z`, is a local homeomorphism.

When `Λ = K₀(D)` (i.e., `v = id`), `V(Σ)` may be a proper subspace of
`Hom(K₀(D), ℂ)`. When `Λ` has finite rank (the case in all modern
applications), `V(Σ) = Hom(Λ, ℂ)` — every homomorphism has finite seminorm
by the support property.

**Corollary 1.3** (Macrì–Stellari, Theorem 4.3): When `Λ` has finite rank,
each connected component of `Stab_Λ(D)` is a complex manifold of dimension
`rk(Λ)`.

### What the literature does

Every modern reference (BMS16, Bayer, Macrì–Stellari lectures, Perry–Pertusi–Zhao,
BLMNPS) defines the seminorm, topology, and local homeomorphism DIRECTLY on
`Stab_Λ(D)` — not on `Stab(D)` with restriction. The class map `v` is a
parameter from the start. `Stab(D)` is the `v = id` case.

### What the formalization currently does (the problem)

The formalization defines `stabSeminorm`, `basisNhd`, and the topology on
`StabilityCondition C` (= `WithClassMap C id`), then gives `WithClassMap C v`
an induced topology. All proofs (deformation theory, seminorm comparison,
connected component theory, local homeomorphism) operate at `v = id`. The
general-`v` result is obtained by:

1. Proving the theorem at `v = id`
2. Restricting to `ClassMapStabilityCondition C v` (a subtype of `StabilityCondition C`)
3. Bridging to `WithClassMap C v` via homeomorphisms

This generates ~400 lines of bridge code in NumericalStabilityManifold.lean.
The bridge code is not mathematical content — it is an artifact of building
at the wrong abstraction level.

## What Phase 1 did

**Commit `e384e8a`** on branch `refactor/withclassmap-primary` changed three
definitions in `StabilityCondition/Defs.lean`:

1. `stabSeminorm`: generalized from `(σ : StabilityCondition C) (U : K₀ C →+ ℂ)`
   to `{v : K₀ C →+ Λ} (σ : WithClassMap C v) (U : Λ →+ ℂ)`.
   Formula: `‖U(K₀.of C E)‖ / ‖σ.Z(K₀.of C E)‖` became
   `‖U(v(K₀.of C E))‖ / ‖σ.Z(v(K₀.of C E))‖`.
   At `v = id`: `v(K₀.of C E) = K₀.of C E`, so the formula is identical.

2. `basisNhd`: generalized from `StabilityCondition C` to `WithClassMap C v`.

3. Topology: replaced TWO definitions (direct on `StabilityCondition C` +
   induced on `WithClassMap C v` with a `priority := 100` hack) with ONE
   definition directly on `WithClassMap C v` via `GenerateFrom`.

**Result:** The ENTIRE project (~3500 lines of downstream proofs) compiled
unchanged, except:
- 1 sorry: `continuous_toStabilityCondition` (bridge infrastructure)
- 1 error: `homeomorphClassMapStabilityCondition` (bridge code)

Net: -4 lines. Two topology definitions → one.

### Why the proofs compiled unchanged

The key technical fact: when `v = id`,

    stabSeminorm_new σ U = sup ‖U(id(K₀.of C E))‖ / ‖σ.Z(id(K₀.of C E))‖
                         = sup ‖U(K₀.of C E)‖ / ‖σ.Z(K₀.of C E)‖
                         = stabSeminorm_old σ U

The `id` application reduces definitionally. So every downstream proof
that uses `stabSeminorm C σ U` where `σ : StabilityCondition C` and
`U : K₀ C →+ ℂ` sees EXACTLY the same term after elaboration.

## Phase 2: Generalize all signatures

### The prediction and its justification

All declarations in `Deformation.lean`, `Topology.lean`, and
`LocalHomeomorphism.lean` currently take `σ : StabilityCondition C`
(= `WithClassMap C id`) and `U : K₀ C →+ ℂ` (= `Λ →+ ℂ` at `v = id`).

**Original prediction:** changing these signatures requires ZERO proof changes.

**What actually happened:** The signature changes are mechanical, but ~75
proof-internal occurrences of `σ.Z (K₀.of C E)` need updating (the charge
applied to a K₀-class). At general `v`, this is `σ.Z (v (K₀.of C E))`.
Additionally, `K₀.of_triangle + map_add` chains need doubling for the
K₀ → Λ → ℂ composition.

**Revised approach: introduce `cl` first.** Rather than mechanically inserting
`v` into 1000+ locations (which makes the code MORE verbose), define
`cl v E := v (K₀.of C E)` ("the class of E in Λ") with a matching API
(`cl_triangle`, `cl_isZero`, `cl_postnikovTower_eq_sum`, etc.) in
`GrothendieckGroup/Basic.lean`. Then the proof changes become SHORTER:
`σ.Z (K₀.of C E)` → `σ.Z (cl v E)`, and the double-map chains become
single lemma applications.

See `artifacts/potential-abstractions-diary.md` Entry 3 for full design.

The prediction was partially wrong: proofs DO access `σ.Z` at the K₀ level
directly (not just through `σ.compat`). But the fix is an abstraction (`cl`),
not a mechanical patch.

### Specific declarations

**Deformation.lean** (~850 lines): All of `linearInterpolationZ`,
`stabSeminorm_smul/add_le/neg`, `stabSeminorm_dominated_of_basisNhd`,
`basisNhd_self/mono`, `exists_basisNhd_subset_basisNhd`, `eq_of_same_Z_near`,
`basisNhd_subset_connectedComponent_small`,
`exists_eq_Z_and_mem_basisNhd_of_stabSeminorm_lt_sin` (the deformation theorem),
`exists_local_lift_sameComponent_in_basisNhd`.

**Why injectivity works at general `v`:** `eq_of_same_Z_near` takes
`hZ : σ.Z = τ.Z`. At `v = id`, this is equality of `K₀ C →+ ℂ` charges.
At general `v`, this is equality of `Λ →+ ℂ` charges — a STRONGER
hypothesis (it implies K₀-charge equality via composition with `v`, but
not vice versa unless `v` is surjective). The proof uses `hZ` only through
`σ.Z(v(K₀.of C E)) = τ.Z(v(K₀.of C E))` (for the phase argument) and
`WithClassMap.ext` (for the conclusion). Both work with `Λ →+ ℂ` equality.
No surjectivity of `v` needed.

**Why the deformation theorem works at general `v`:** The theorem produces
`τ : WithClassMap C v` with `τ.Z = W` (a prescribed charge) and
`τ ∈ basisNhd C σ ε`. The proof constructs a slicing and pairs it with `W`
as the charge. At `v = id`, `W : K₀ C →+ ℂ`. At general `v`, `W : Λ →+ ℂ`.
The construction accesses `W` only through `stabSeminorm` (now at the `v`
level) and the compatibility condition. The proof builds a `WithClassMap C v`
directly — there is no intermediate `StabilityCondition C` to lift from.

**Topology.lean** (~710 lines): `bridgeland_lemma_6_4`, `P_of_Q_of_P_semistable`,
`phiMinus_le_le_phiPlus`, `bridgeland_6_4_one_dir`, and all supporting lemmas.

Also: DELETE `StabilityCondition.CentralChargeIsLocalHomeomorphOnConnectedComponents`

Also: `continuous_toStabilityCondition` — assess after Phase 2. If

topology (because `toStab⁻¹(basisNhd_id C (toStab σ) ε) = basisNhd_v C σ ε`
by definitional equality of the seminorm), plus `basisNhd` containment for

**LocalHomeomorphism.lean** (~650 lines): `componentRep`,

## Risk assessment

**Low risk:** Phases 2a-2c (signature generalization). The proofs compiled
at `v = id` after Phase 1. Changing signatures should be mechanical.

**Medium risk:** Phase 3 (NumericalStabilityManifold.lean rewrite). The
deletion is large and the rewritten manifold theorem needs to compose
`ComponentTopologicalLinearLocalModel` with finite-dimensionality. This is
new code (though short).

**If something breaks unexpectedly:** STOP. Read the error. The error tells
us what the proof secretly depends on. That's the most valuable output of
the refactor — it reveals hidden assumptions. Do NOT patch around it.
